Task 2

Unity NavMesh is the pathfinding and spatial reasoning system built within the engine. There are several components inside this system such as NavMeshAgent, NaveMeshSurface and NavMeshObstacle, each of them having different roles.


NavMeshSurface creates a walkable surface on a mesh (plane, stairs, etc) which is further used by the NavMeshAgent. I had the chance to study the code behind their navigation system, however, Unity’s API is built on layers upon layers of abstraction. It is difficult to see what happens at the core. My assumption is that this system takes in the user input such as the objects in the scene to consider, the agent properties (type, radius, height), Layers to ignore and with all this data it calculates the world bounds and builds a collection of convex polygons interconnected.

The NavMeshAgent is an algorithm which displays steering behaviours in order to make the character move autonomously and realistically on the walkable surface (NavMeshSurface). The pathfinding algorithm used in this agent is A* which at the core uses Dijkstra algorithm to traverse the graph built by the NavMesh. The reason why it builds a collection of convex polygons is that it only needs to traverse one polygon and the rest are discarded if the start and destination are in the same area (graph). After choosing the shortest path, the trajectory is smoothed out by having a larger radius for each node. This agent also has collision detection implemented which determines how the AI should react when it encounters a non-stationary object.

NavMeshObstacles are basically box colliders which the NavMeshAgent can understand. It is very useful when you have objects that can be move around and the AI has to avoid them. These objects cannot be baked into the NavMeshSurface because it will leave an empty space where the agent won’t be able to walk.

I am sure there are more elaborate calculations and algorithms at the core of this system, and this is just an overview of what I believe the Unity Navigation system represents.
