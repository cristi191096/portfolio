Multiplayer fast-pace action game

Battle Of The Stars

This game was part of a design module during the second year that allowed us to put our best ideas in a single project with no restriction. I had many concepts in my mind, but I wanted to experience something new. Making a multiplayer game came as a wonderful idea so I started doing research on this. In a multiplayer game, your character is not transferred into another computer, there are copies of you instance made in the other players’ devices and all it’s sent across the network are bytes of data that can represent anything, for example: position in game world, items used, is the player attacking or having any special behaviour, damage given/taken.

Unity provides us with a brilliant service for making a multiplayer game. They can host a dedicated server if needed and have two types of API. Low Level API is much more flexible, but requires more coding and time to make it work. The High Level API is fairly easy to use, however, it cannot offer the same flexibility. I used both in this project. Well, I tried them both. I started with the Low Level API. Although it was fun and I could have learnt a lot, I realised I don’t have enough time to finish the game in this way, so I switched to the High Level API.
